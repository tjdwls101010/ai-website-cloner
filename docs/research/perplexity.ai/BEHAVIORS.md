# Perplexity Behaviors

Target pages:
- Home: `https://www.perplexity.ai/`
- Conversation: generated by submitting `What are the latest AI search trends?`, observed at `/search/199ef299-d83d-4de9-b3ba-9e14e979ee4d`

## Global Interaction Model
- Desktop uses a fixed 200px left sidebar and a main content area.
- Mobile replaces the sidebar with a top bar containing menu, logo, Business Info, and Open in App.
- Home page is mostly static. The central composer accepts text and submits to a conversation page.
- Conversation page is scrollable and keeps the follow-up composer pinned near the lower viewport.
- Cookie banner appears as a fixed lower-right card on desktop and a full-width bottom card on mobile.

## Observed States
- Sidebar item hover: background changes to subtle sand `rgba(39, 26, 0, 0.07)` with rounded 12px container.
- Active sidebar item: New uses subtle background on home; current thread uses subtle pill on conversation.
- Composer focus: rounded white/sand container keeps a 1px border and soft shadow, text caret in dark foreground.
- Conversation tabs: active Answer tab has a dark bottom border; Links and Images are quiet.
- Cookie buttons: primary dark button, secondary pale sand button.

## Responsive Behavior
- Desktop `>= 768px`: left sidebar visible, main content starts at x=200px.
- Mobile `< 768px`: sidebar hidden, mobile top bar visible, content centered in a single column.
- Mobile home title changes from wordmark-only desktop treatment to large question text: `What do you want to know?`
- Mobile cookie banner occupies almost the full bottom width with large 28px body text.

## Known Capture Notes
- A second clean-cookie capture hit Cloudflare verification, so the first successful browser captures are the source of truth.
- The clone intentionally has no live backend, authentication, account capture, or real Perplexity service calls.
